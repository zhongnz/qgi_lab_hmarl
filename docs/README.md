# Documentation Hub

Use this folder for all non-code project documentation.

## Structure

- `meetings/`:
  Meeting notes and minutes.
- `decisions/`:
  Architecture and project decisions (ADR-style records).
- `architecture/`:
  Working design notes and diagrams from technical reviews.
- `reports/`:
  Experiment summaries, weekly reports, and milestone writeups.
- `templates/`:
  Reusable markdown templates.

## Conventions

1. Meeting file names:
   `YYYY-MM-DD_topic.md`  
   Example: `2026-02-20_weekly-sync.md`
2. Decision file names:
   `ADR-XXXX_title.md`  
   Example: `ADR-0001_mappo_ctde_choice.md`
3. Keep entries short and actionable:
   - decisions made
   - open questions
   - owners and due dates
